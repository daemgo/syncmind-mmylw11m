### 信息采集流程（所有路径共用）

**阶段 0：企业预扫描（必须按顺序完成，不可跳过）**

步骤 0.1 — 基础信息搜索，依次执行：
1. "{company_name} 上市 股票代码"
2. "{company_name} 融资 轮次 估值"
3. "{company_name} 官网"
4. "{company_name} 企查查 天眼查"

步骤 0.2 — 官网探测（搜索引擎优先，域名推测仅作 fallback）：
- 搜索引擎：`"{company_short_name} 官网 site:.com OR site:.cn OR site:.tech"`
- 域名推测 fallback（仅当搜索引擎无结果时，最多尝试 3 个最可能的域名）：提取品牌名/关键词，拼接 `.com` / `.cn` / `.com.cn`，用 WebFetch 验证

步骤 0.3 — 官网内容深挖（找到官网则必须执行）：

| 页面 | 提取内容 | 分析要点 |
|------|----------|----------|
| 首页 | 公司定位、核心业务、Slogan | 判断行业、目标客户、价值主张 |
| 产品/服务页 | 产品线、目标客户、技术特点 | 判断细分领域、技术能力 |
| 关于我们 | 公司规模、发展历程、团队背景 | 判断阶段、团队能力 |
| 新闻/动态 | 最新动向、融资、合作、获奖 | 判断战略方向、政策信号 |
| 招聘/加入我们 | 技术栈、团队规模、岗位结构 | 推断技术方向、扩张速度 |
| 客户案例 | 客户类型、行业分布 | 判断市场定位 |
| 荣誉资质 | 专精特新、小巨人、高新技术企业等 | 政策信号分析 |

步骤 0.4 — 中国特色数据源搜索（必须执行）：
```
工商信息（三选一，按优先级）：
1. "{company_name} 企查查"
2. "{company_name} 天眼查"
3. "{company_name} 爱企查"

政策信号：
1. "{company_name} 专精特新 小巨人"
2. "{company_name} 瞪羚企业 独角兽"
3. "{company_name} 数字化转型 智能制造"
4. "{company_name} 环保 ESG 绿色制造"

司法风险：
1. "{company_name} 诉讼 裁判文书网"
2. "{company_name} 行政处罚 环保 税务"

招投标（政企/国企适用）：
1. "{company_name} 招投标 招标网"
2. "{company_name} 政府采购"

行业/地域：
1. "{company_name} 行业排名 市场份额"
2. "{company_name} {城市} 产业园"
```

步骤 0.5 — 补充信息搜索：
```
1. "{company_name} 创始人 CEO 董事长 团队"
2. "{company_name} 客户 案例 合作伙伴"
3. "{company_name} 招聘 boss直聘 拉勾 猎聘"
4. "{company_name} 知乎 脉脉 评价 口碑"
5. "{company_name} 竞争对手 竞品 市场份额"
6. "{product_name} 评测 对比"（如已知产品名）
7. "{company_name} 新闻 动态 大事件"
```

步骤 0.6 — 路径判断：

| 条件 | 路径 |
|------|------|
| 有股票代码（A股/港股/美股/新三板） | **Path A**（数据驱动） |
| 无上市但：员工 > 500人 / 融资C轮+ / 年营收 > 5亿 / 知名度高 | **Path A**（数据驱动） |
| 有官网且信息丰富（产品清晰、有客户案例） | **Path B+**（行业驱动+官网洞察） |
| 官网简陋或无官网 | **Path B**（纯行业驱动） |

输出判断结果：

### Path A：数据驱动分析（上市/大型企业）

适用于：上市公司、大型企业、知名独角兽、国企。在共用分析模块基础上增加以下分析：

**A1 — 多维数据深度扫描**

| 维度 | 数据来源 | 搜索策略 | 分析目标 |
|------|----------|----------|----------|
| 财务与战略 | 年报、季报、业绩快报 | 搜索"年报 季报 业绩快报" | 判断扩张/收缩/转型期 |
| 舆情体验 | 微博、知乎、脉脉、黑猫投诉 | 搜索"评价 口碑 投诉" | 挖掘用户/员工痛点 |
| 行业地位 | 行业报告、排名 | 搜索"行业排名 市场份额" | 判断竞争压力 |
| 招投标 | 招标网 | 搜索"招投标 招标网" | 采购意向和预算信号 |

**A2 — 财务/融资信号解读**

| 信号类型 | 判断标准 | 销售含义 |
|----------|----------|----------|
| 新一轮融资后 | 新闻显示近期融资 | 刚拿钱，有预算，决策可能变快 |
| 融资轮次间隔变长 | 融资间隔 > 12个月 | 资金紧张，需要精打细算 |
| 营收增长但利润下降 | 财报数据 | 扩张期，可能需要提效工具 |
| 大额应收账款 | 财报数据 | 现金流压力，可推应收管理系统 |
| 现金流充裕 | 财报数据 | 预算充足，可推高端方案 |
| 大额投资扩产 | 新闻/公告 | 扩张期，需要管理系统 |
| 裁员/收缩 | 新闻/公告 | 需要降本增效工具 |
| 子公司/并购 | 工商/新闻 | 整合需求强 |

**A3 — 企业文化/价值观信号**

| 文化信号 | 判断依据 | 销售含义 |
|----------|----------|----------|
| 技术导向 | 大量技术岗位、技术博客、专利 | 推技术领先方案 |
| 销售导向 | 大量销售岗位、销售口号 | 推销售赋能工具 |
| 客户导向 | 强调服务、客户案例 | 推客户管理/服务系统 |
| 成本导向 | 强调性价比 | 推降本增效方案 |

**A4 — 竞争对手/生态位分析**

| 分析维度 | 搜索关键词 | 销售含义 |
|----------|------------|----------|
| 上下游关系 | "供应商 合作伙伴 客户案例" | 可能有间接影响力 |
| 生态位 | 行业分析报告 | 平台型/工具型/服务型？定位不同话术不同 |
| 行业排名 | "行业排名 市场份额 Top" | 头部/腰部/尾部的诉求不同 |
| 主要竞对 | "竞争对手 竞品" | 差异化机会在哪里？ |

**A5 — 招投标信息（政企/国企）**

| 搜索目标 | 搜索关键词 | 销售含义 |
|----------|------------|----------|
| 招标历史 | "招投标 招标网 政府采购" | 采购偏好、预算规模 |
| 中标结果 | "中标 成交" | 竞争对手、中标价格 |
| 招标预告 | "招标预告 招标计划" | 潜在机会 |

### Path B/B+：行业驱动分析（中小型企业）

适用于：非上市中小企业、传统制造业、区域性企业。
- Path B+：有官网，从官网提取产品/业务信息，痛点分析更精准
- Path B：无官网或官网无有效信息，纯靠行业痛点库推断

在共用分析模块基础上增加以下分析：

**B1 — 行业痛点库匹配**

根据企业所属行业，从 `industry-pain-points.md` 中匹配对应痛点库，结合企业实际情况分析数字化机会。
- Path B+：基于官网信息做精准痛点匹配
- Path B：使用行业通用痛点 + 规模假设

**B2 — 技术栈推断**

有官网/招聘信息时：从招聘 JD、产品形态、官网技术实现推断。

无信息时，使用通用假设：

| 企业规模 | 典型技术栈现状 | 销售对策 |
|----------|----------------|----------|
| 微型（<20人） | Excel + 微信 + 简单工具 | 推极简SaaS、强调易用性 |
| 小型（20-50人） | Excel + 微信 + 简单财务软件 | 推轻量级SaaS、快速上线 |
| 中型（50-200人） | 单机版ERP或初级云ERP + Excel | 推一体化方案、数据打通 |
| 中大型（200-500人） | 传统ERP（用友/金蝶）+ 多个独立系统 | 推集成平台、数据中台 |
| 科技公司 | 自研系统 + 通用SaaS（钉钉/飞书） | 推API优先、可扩展方案 |
| 传统企业 | 传统ERP + Excel + 微信 | 推数字化升级、强调提效 |

**B3 — 生成访谈提纲**
- Path B+：基于官网信息的针对性访谈提纲
- Path B：行业通用访谈提纲（必选），帮助销售在首次拜访中验证假设

### 共用分析模块（Path A 和 Path B/B+ 都执行）

**C1 — 工商信息解读**

从企查查/天眼查提取并解读：

| 信息项 | 解读规则 | 销售含义 |
|--------|----------|----------|
| 注册资本 | 高注册资本=资金充足 | 预算可能更高 |
| 实缴资本 | 实缴/注册资本比例 | 资金真实性判断 |
| 成立时间 | 成立时间短=初创/新业务 | 决策链条可能较短 |
| 股东信息 | 股东类型（个人/法人/国企） | 个人=家族企业；法人=职业化；国企=决策链长 |
| 分支机构 | 数量与地域分布 | 分支多=扩张期/多区域运营 |
| 司法风险 | 诉讼数量/类型、被执行人 | 法律/资金链风险 |
| 经营异常 | 异常记录 | 公司状态不稳定 |
| 行政处罚 | 环保/税务/市场监管处罚 | 合规风险 |
| 变更记录 | 法人/地址频繁变更 | 公司不稳定 |

**C2 — 政策敏感度分析**

| 政策信号 | 搜索关键词 | 销售含义 |
|----------|------------|----------|
| 专精特新 | "专精特新 小巨人" | 有政策资金支持，预算更足 |
| 数字化转型 | "数字化转型 智能制造 工业互联网" | 有政策推动，IT预算充足 |
| 环保/ESG | "环保 绿色制造 碳中和 ESG" | 需要环保合规系统 |
| 信创 | "信创 国产化 自主可控" | 必须支持国产软硬件 |
| 高新技术企业 | "高新技术企业" | 有税收优惠，预算更灵活 |
| 瞪羚/独角兽 | "瞪羚企业 独角兽" | 成长期，扩张需求强 |

**C3 — 组织架构/决策链条推断**

| 组织特征 | 判断依据 | 销售含义 |
|----------|----------|----------|
| 家族企业 | 股东为个人，股权集中 | 家长式决策，搞定关键人即可 |
| 职业经理人 | 股东为法人，招聘高管 | 决策流程化，需要多方说服 |
| 国企/央企 | 股东为国企/政府 | 决策链条长，需要招标流程 |
| 集团子公司 | 股权穿透有集团 | 可能需要集团审批 |
| 上市公司 | 有股票代码 | 决策流程化，需要合规 |

**C4 — 时机判断**

| 时机类型 | 判断标准 | 销售策略 |
|----------|----------|----------|
| 刚融资/刚上市 | 近期融资/上市 | 追热点，预算充足，快速切入 |
| 业务扩张期 | 大量招人、开新店/新厂 | 强调效率、规模化支持 |
| 转型阵痛期 | 裁员、业务调整、战略转型 | 强调ROI、快速见效 |
| 稳定期 | 增长平缓 | 强调长期价值、稳定可靠 |
| 危机期 | 大量负面新闻、司法风险 | 避免进入或谨慎观望 |

**C5 — 信用/风险信号检查**

| 风险类型 | 检查方法 | 销售含义 |
|----------|----------|----------|
| 司法诉讼 | 搜索"诉讼 裁判文书网" | 法律风险，可能影响付款 |
| 经营异常 | 工商信息异常 | 公司状态不稳定 |
| 行政处罚 | 环保、税务处罚 | 合规风险 |
| 股东冻结 | 股权冻结 | 资金链有问题 |
| 频繁变更 | 法人/地址频繁变更 | 公司不稳定 |

**C6 — 招聘信息解读**

| 招聘岗位 | 推断结论 | 销售对策 |
|----------|----------|----------|
| 大量 Java/Go 开发 | 自研能力强 | 推 PaaS/中间件/API 集成方案 |
| 大量 ERP/SAP 运维 | 核心系统沉重 | 推自动化/低代码/集成方案 |
| 大量 数据分析师/算法 | 数字化转型深水区 | 推数据中台/BI/数据治理 |
| 大量 客服/运营 | 服务压力大 | 推智能客服/知识库/工单系统 |
| 大量 供应链岗位 | 供应链是痛点 | 推 SCM/WMS/供应商管理 |
| 大量 销售/BD | 拓客压力 | 推 CRM/销售自动化 |

**C7 — 地域/产业带特色分析**

| 产业带 | 地域 | 特色痛点 |
|--------|------|----------|
| 外贸集群 | 义乌、深圳、宁波 | 跨境电商、汇率、海外仓、合规 |
| 制造业集群 | 苏州、东莞、佛山 | 供应链协同、环保合规、自动化 |
| 互联网集群 | 北京海淀、杭州、深圳 | 招人难、快速迭代、技术创新 |
| 服务业集群 | 各省会 | 本地化服务、会员运营、复购 |

### 降级策略

当搜索工具受限（限流、无结果）时：
1. 基于已有信息做最大化推演，明确标注"基于有限信息推断"
2. 降低 `metadata.confidence`：搜索完整=80-100，部分缺失=50-79，大量缺失=20-49
3. 在档案中标注信息缺口，建议销售在拜访时验证
4. 不因信息不足而放弃输出——有限信息的推演档案 >> 没有档案

### 输出规则

**双重输出**：完成分析后执行两步：
1. **对话展示**：以 Markdown 格式展示完整档案
2. **写入 JSON**：将结构化数据写入 `docs/customer/profile.json`（类型定义见 `apps/platform/types/customer-profile.ts` 中的 `CustomerFullProfile`）

**JSON 字段映射**：

| 分析模块 | 写入字段 | 说明 |
|----------|----------|------|
| 企业类型判断 | `companyName`, `shortName`, `isListed`, `stockCode`, `website`, `industry`, `subIndustry`, `scale`, `mainBusiness`, `products`, `targetCustomers`, `businessModel`, `rating`, `tags` | 基础信息 |
| AI 一句话摘要 | `summary` | 如"民企 · 制造业/汽车零部件 · 中型 · 扩张期" |
| 分析路径 | `strategy.analysisPath`, `strategy.informationQuality` | Path A/B+/B |
| 工商信息（C1） | `registration.*` | 注册资本、股东、分支机构等 |
| 政策分析（C2） | `policyAnalysis.*` | 专精特新、数字化转型等 |
| 组织架构（C3） | `organization.*` | 组织类型、决策链条、关键人 |
| 时机判断（C4） | `timing.*` | 所处时机、判断依据、切入策略 |
| 风险信号（C5） | `creditRisk.*` | 司法风险、经营异常等 |
| 战略态势 | `strategy.*` | 增长阶段、竞争位置、数字化成熟度 |
| 技术栈 | `techStack.*` | 当前系统、技术能力、数字化缺口 |
| 关键数据 | `keyMetrics` | `{ label, value, trend }`，trend 无数据时为 null |
| 信息来源 | `metadata.sources` | 所有参考链接 |

**写入规则**：
- 只写 profile 负责的字段，不触碰其他 skill 的文件
- merge 写入：保留 JSON 中已有的其他字段值，只更新 profile 负责的字段
- `metadata.createdAt`：首次生成时写入当前时间
- `metadata.updatedAt`：每次更新时写入当前时间
- `metadata.createdBy`：设为 `"AI"`
- `metadata.confidence`：根据信息丰富度设置（高=80-100, 中=50-79, 低=20-49）

**版本管理**：

| 场景 | 版本变化 | 操作 |
|------|----------|------|
| 首次生成 | `"1.0"` | 全量搜索 + 全量写入 |
| 用户要求"更新档案" | minor +0.1（如 1.0→1.1） | 重新执行阶段0搜索，仅更新有变化的字段 |
| 企业发生重大变化（上市、融资、并购、行业变更） | major +1.0（如 1.1→2.0） | 全量重新分析 |

如果新搜索结果导致 `timing.phase`、`scale`、`isListed`、`industry` 等核心字段变化，视为 major update。

**不写入的字段**：

| 字段 | 由谁负责 |
|------|----------|
| `relationship` | 平台 CRM / 项目系统 |
| 客户需求 | `/requirements` |
| 销售作战 | `/sales-guide` |

### 启动程序

如果用户未提供企业名称，询问："请提供您想要分析的目标企业名称。"

收到企业名称后立即执行：
1. 执行阶段 0（全部步骤，不可跳过）
2. 判断路径，执行对应分析：
   - Path A → 共用分析模块 + Path A 专属分析
   - Path B+/B → 共用分析模块 + Path B 专属分析
3. 按输出规则执行双重输出（Markdown 展示 + JSON 写入）

分析质量要求（内嵌于每个分析步骤）：
- 推演结论，不复述事实：不写"他们在招人"，写"大量招聘数据工程师，推断正在建数据中台"
- 信息不足时基于行业/区域假设推演，标注"假设待验证"，不因缺少信息放弃输出
- 负面信号（诉讼、裁员、差评）往往藏着销售机会，必须分析其销售含义
